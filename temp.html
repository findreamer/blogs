1.
Amis 结构及组成​
Amis 低代码平台从功能方面主要划分为两个核心模块，分别是页面渲染，和Schema 编辑器。整个amis 由4个子包组成，以下为各个子包主要功能：​
•
amis：amis-core + amis-ui + 注册组件渲染器​
•
amis-core：渲染器核心逻辑 + 方法 + 类型​
•
amis-editor：amis-editor-core + 注册编辑器插件 +注册渲染控制器​
•
amis-editor-core：编辑器基础功能​
2.
渲染原理​
2.1
render 函数​
amis 的渲染方式是调用其对外暴露的 render 方法，将schema渲染成组件，我们查找源码找到其所在位置。​
由上图可知，render函数路径是由amis-core/src/index.tsx文件导出的，其内部返回一个函数式组件AMISRenderer，并将接收的参数透传给该组件，接下来去 AMISRenderer 组件看下。​
2.2
AMISRenderer 组件​
直接看组件源码：​
由上图可知，该组件实现以下的大致逻辑：​
1.
初始化默认语言。​
2.
初始化/更新全局store。​
3.
初始化schema，基于语言、替换规则对schema做处理。​
4.
渲染根组件ScopedRootRenderer。​
​
2.2.1
初始化 store 的过程：​
在文件packages\amis-core\src\factory.tsx中定义了全局的 store。 ​
export const stores: {​
[propName: string]: IRendererStore;​
} = {};​
回到 packages\amis-core\src\index.tsx 文件，store 通过 useMemo 创建，当 options 发生变化时会更新 store。​
import { stores } from './factory'​
​
function AMISRenderer({ schema, options, pathPrefix, ...props }) {​
​
// 省略部分代码​
const store = React.useMemo(() => {​
// 从全局stores map 对象中取出 store，默认取global​
let store = stores[options.session || 'global'];​
​
// 如果不存在store，则通过 RendererStore.create 创建 store​
if (!store) {​
options = {​
...defaultOptions,​
...options,​
fetcher: options.fetcher​
? wrapFetcher(options.fetcher, options.tracker)​
: defaultOptions.fetcher,​
confirm: promisify(​
options.confirm || defaultOptions.confirm || window.confirm​
),​
locale,​
translate​
} as any;​
​
store = RendererStore.create({}, options);​
stores[options.session || 'global'] = store;​
} else {​
// 更新 env​
const env = getEnv(store);​
Object.assign(env, {​
...options,​
fetcher: options.fetcher​
? wrapFetcher(options.fetcher, options.tracker)​
: env.fetcher,​
confirm: options.confirm ? promisify(options.confirm) : env.confirm,​
locale,​
translate​
});​
}​
​
(window as any).amisStore = store; // 为了方便 debug.​
return store;​
}, Object.keys(options).concat(Object.values(options)).concat(locale));​
​
// 省略部分代码​
// ...​
}​
